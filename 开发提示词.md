# 数据备份软件开发提示词

## 项目定位
开发一款 Windows 环境下的数据备份软件，使用 C++17 + Qt + CMake，采用策略模式和面向对象设计。

## 核心技术栈
- 语言：C++17（必须使用 `<filesystem>` 处理路径）
- GUI：Qt 6
- 构建：CMake 3.15+
- 测试：Google Test
- 第三方库：zlib、lzma、OpenSSL

## 功能清单（按优先级）

### P0：基础功能（必须完成）
1. **目录树备份**：递归备份指定目录及其所有子文件
2. **数据还原**：将备份文件精确还原到目标位置
3. **元数据保存**：保存文件属性、创建时间、修改时间（使用 Windows API）

### P1：核心扩展（重点完成）
4. **文件过滤**：实现 5 种过滤器（路径、类型、文件名、时间、大小）
5. **文件打包**：将多个文件打包为一个连续文件，提升 I/O 效率
6. **压缩算法**：实现 Zlib 和 LZMA 压缩（策略模式）
7. **加密算法**：实现 AES 和 XOR 加密（策略模式）

### P2：高级功能（加分项）
8. **定时备份**：支持周期性任务调度
9. **实时备份**：使用 `ReadDirectoryChangesW` 监控文件变化
10. **图形界面**：Qt GUI，提供配置、进度、日志显示

## 关键设计模式

### 策略模式（核心架构）
```cpp
// 压缩策略接口
class ICompressionStrategy {
public:
    virtual bool compress(const std::vector<char>& input, std::vector<char>& output) = 0;
    virtual bool decompress(const std::vector<char>& input, std::vector<char>& output) = 0;
    virtual std::string getName() const = 0;
};

// 加密策略接口
class IEncryptionStrategy {
public:
    virtual bool encrypt(const std::vector<char>& plaintext, std::vector<char>& ciphertext, const std::string& password) = 0;
    virtual bool decrypt(const std::vector<char>& ciphertext, std::vector<char>& plaintext, const std::string& password) = 0;
};

// 文件过滤器接口
class IFileFilter {
public:
    virtual bool shouldInclude(const fs::path& filePath) const = 0;
};
```

```cpp
class FilterChain : public IFileFilter {
```

## 测试驱动开发（TDD）原则

### 核心原则
1. **先写测试，再写代码**：每个功能模块开发前，先编写对应的测试用例
2. **测试先行**：定义接口后立即编写测试框架，确保接口可测试
3. **持续验证**：每完成一个模块立即运行测试，不等到最后集成测试
4. **覆盖率要求**：每个模块测试覆盖率 ≥ 80%，关键算法模块要求 100% 分支覆盖
5. **回归测试**：每次修改后必须运行全部测试，确保不破坏已有功能

### 测试策略
- **单元测试**：针对每个类和函数的独立测试
- **集成测试**：模块间协作的功能验证
- **端到端测试**：完整的备份-还原流程测试

**问题描述**：
```
CMake Error: The C++ compiler is not able to compile a simple test program.
```

**原因**：
- CMake 默认使用 MinGW Makefiles，但 MSVC cl.exe 不兼容 MinGW 的 make
- 路径编码问题导致中文乱码

**避免方法**：
- ✅ **必须使用 Visual Studio 2022 生成器**
- ✅ 在 CMakeLists.txt 中强制指定生成器
- ❌ 不要使用 MinGW Makefiles + MSVC 混合模式

**正确做法**：
```powershell
# 在 build_project.bat 中强制使用 VS 2022
cmake .. -G "Visual Studio 17 2022" -A x64
```

### 问题 5：FileInfo 类设计不一致（最重要）

**问题描述**：
```cpp
// 编译错误
class "FileInfo" 没有成员 "path"
class "FileInfo" 没有成员 "size"
```

**原因**：
- FileInfo.h 使用 getter/setter 方法（getPath(), getSize()）
- MetadataManager 和 FileScanner 尝试直接访问成员变量
- 接口设计不一致导致编译失败

**避免方法**：
- ✅ **使用公共成员变量**，便于直接访问和序列化
- ✅ 避免使用 getter/setter 方法（增加复杂度）
- ✅ 成员变量命名直接使用（path, size, isHardlink）

**正确做法**：
```cpp
// FileInfo.h - 正确设计
class FileInfo {
public:
    fs::path path;                    // 公共成员变量
    uint64_t size = 0;
    bool isDirectory = false;
    bool isHardlink = false;
    bool isSymlink = false;
    uint64_t creationTime = 0;
    uint64_t lastWriteTime = 0;
    uint64_t lastAccessTime = 0;
    DWORD attributes = FILE_ATTRIBUTE_NORMAL;
    fs::path hardlinkTarget;
    uint32_t hardlinkCount = 0;
    fs::path symlinkTarget;
    std::string permissions;

    FileInfo() = default;
    explicit FileInfo(const fs::path& filePath) : path(filePath) {}
};

// 使用方式
FileInfo info;
info.path = "test.txt";       // 直接访问
info.size = 1024;              // 直接设置
EXPECT_EQ(info.path, "test.txt");  // 直接读取
```

### 问题 9：时间戳类型不匹配

**问题描述**：

```cpp
ULARGE_INTEGER uli;
uli.LowPart = filetime.dwLowDateTime;
uli.HighPart = filetime.dwHighDateTime;
uint64_t timestamp = uli.QuadPart;

// uint64_t 转 FILETIME
ULARGE_INTEGER uli;
uli.QuadPart = timestamp;
FILETIME filetime;
filetime.dwLowDateTime = uli.LowPart;
filetime.dwHighDateTime = uli.HighPart;
```

## 核心实现要点

### 文件扫描（含链接文件处理）
```cpp
// 使用 std::filesystem 递归遍历，处理硬链接和软链接
class FileScanner {
private:
    std::unordered_map<std::pair<uint64_t, uint64_t>, fs::path, pair_hash> hardlinks_;
    
public:
    void scan(const fs::path& rootPath) {
        // 广度优先遍历，避免递归栈溢出
        std::queue<fs::path> dirQueue;
        dirQueue.push(rootPath);
        
        while (!dirQueue.empty()) {
            auto currentDir = dirQueue.front();
            dirQueue.pop();
            
            for (auto& entry : fs::directory_iterator(currentDir)) {
                // 处理软链接
                if (entry.is_symlink()) {
                    if (shouldFollowSymlinks_) {
                        // 解析符号链接指向的实际文件
                        auto target = fs::read_symlink(entry.path());
                        // 继续处理目标文件...
                    }
                    continue;
                }
                
                // 处理硬链接（仅限 Windows）
                if (entry.is_regular_file()) {
                    HANDLE hFile = CreateFileW(
                        entry.path().c_str(),
                        0, FILE_SHARE_READ | FILE_SHARE_WRITE,
                        nullptr, OPEN_EXISTING,
                        FILE_FLAG_OPEN_REPARSE_POINT, nullptr
                    );
                    
                    BY_HANDLE_FILE_INFORMATION fileInfo;
                    GetFileInformationByHandle(hFile, &fileInfo);
                    CloseHandle(hFile);
                    
                    // 使用文件索引作为唯一标识
                    uint64_t fileIndex = (uint64_t(fileInfo.nFileIndexHigh) << 32) | fileInfo.nFileIndexLow;
                    uint64_t volumeSerial = fileInfo.dwVolumeSerialNumber;
                    
                    auto key = std::make_pair(volumeSerial, fileIndex);
                    
                    // 检查是否已遇到该硬链接
                    if (hardlinks_.find(key) != hardlinks_.end()) {
                        // 这是硬链接的重复引用，记录但不重复备份
                        FileInfo info;
                        info.path = entry.path();
                        info.isHardlink = true;
                        info.hardlinkTarget = hardlinks_[key];
                        continue;
                    }
                    
                    // 首次遇到该文件，记录引用计数
                    hardlinks_[key] = entry.path();
                    
                    FileInfo info;
                    info.path = entry.path();
                    info.size = entry.file_size();
                    info.hardlinkCount = fileInfo.nNumberOfLinks;
                    // 提取其他元数据...
                }
                
                if (entry.is_directory()) {
                    dirQueue.push(entry.path());
                }
            }
        }
    }
};
```

## 异常处理原则
- 捕获所有异常，不崩溃
- 文件被占用：跳过并记录日志
- 磁盘空间不足：提前终止并提示
- 权限不足：跳过并记录
- 继续执行其他任务，不因单文件失败终止

## 安全性要求
- 密钥不在日志中明文输出
- 使用 PBKDF2 从密码派生密钥
- 密钥使用后及时清零
- 错误密码无法解密

## 测试要求
- 每个算法类和过滤器类都要有单元测试
- 使用 Google Test 框架
- 测试正常情况和异常情况
- 内存检测无泄漏
